LIBRARY MANAGEMENT SYSTEM - TEST SCENARIOS AND EXAMPLES

This document provides detailed test scenarios and expected outputs for all features.


TEST SCENARIO 1: ADD BOOKS TO LIBRARY
=====================================

Objective: Add multiple books to the library system

Test Case 1.1 - Add First Book
Steps:
1. Select option 1 (Add New Book)
2. Enter Title: "The Great Gatsby"
3. Enter Author: "F. Scott Fitzgerald"
4. Enter ISBN: "978-0-7432-7356-5"
5. Enter Quantity: 5
6. Enter Price: 250

Expected Result:
- Book Added Successfully message
- Book ID: 1001 assigned
- 5 copies available in library

Validation:
- Use option 2 to verify book appears in list


Test Case 1.2 - Add Multiple Books
Steps:
1. Add Book 2: "1984" by George Orwell, ISBN 978-0-451-52493-2, 3 copies, 300 Rs
2. Add Book 3: "To Kill a Mockingbird" by Harper Lee, ISBN 978-0-06-112008-4, 4 copies, 280 Rs
3. Add Book 4: "Pride and Prejudice" by Jane Austen, ISBN 978-0-14-143951-8, 6 copies, 200 Rs

Expected Book IDs:
- "1984" - Book ID: 1002
- "To Kill a Mockingbird" - Book ID: 1003
- "Pride and Prejudice" - Book ID: 1004

Validation:
- All books appear in Display All Books
- Availability shows total and available quantities

TEST SCENARIO 4: ISSUE BOOKS
=============================

Test Case 4.1 - Simple Book Issue
Objective: Issue one book to member

Steps:
1. Select option 7 (Issue Book)
2. Enter Member ID: 2001 (Alice Johnson)
3. Enter Book ID: 1001 (The Great Gatsby)

Expected Result:
- Book Issued Successfully message
- Issue Record ID: 5001
- Due Date: 14 days from today
- Alice's books issued count: 1/3

Validation:
- Book availability changes from 5/5 to 4/5
- Member books issued changes from 0/3 to 1/3


Test Case 4.2 - Multiple Issues to Same Member
Objective: Issue multiple books to one member (within limit)

Steps:
1. Issue Book 1002 to Member 2001 (Alice) - Record ID: 5002
2. Issue Book 1003 to Member 2001 (Alice) - Record ID: 5003

Expected Result:
- Both issues successful
- Alice now has 3/3 books
- Each book gets unique issue record ID
- Each gets 14-day due date


Test Case 4.3 - Issue Exceeds Limit
Objective: Verify 3-book limit enforcement

Steps:
1. Alice already has 3 books issued
2. Try to issue Book 1004 to Member 2001

Expected Result:
- Message: "Member has already issued 3 books (maximum limit reached)!"
- Book 1004 not issued
- Alice still has 3/3 books


Test Case 4.4 - Issue from Unavailable Book
Objective: Verify unavailable book handling

Prerequisites: Book 1001 has only 5 copies, all 5 already issued

Steps:
1. Try to issue Book 1001 to Member 2002 (Bob Smith)

Expected Result:
- Message: "Book is not available!"
- Issue fails
- Bob still has 0 books

Test Case 5.3 - Return Invalid Record

===================

- 20 books, 10 members: Instant response
- 100 books, 50 members: Smooth operation
- 500 books, 200 members: Noticeable delay on display
- 1000+ records: May need database optimization

DEBUGGING CHECKLIST
===================

If tests fail:
[ ] Check files exist in correct directory
[ ] Verify read/write permissions
[ ] Delete .dat files and start fresh
[ ] Check member/book IDs are correct
[ ] Verify issue record IDs for returns
[ ] Check date format is correct
[ ] Ensure book has available copies
[ ] Confirm member doesn't exceed 3-book limit
[ ] Verify file corruption didn't occur


SUCCESS CRITERIA
================

All tests pass if:
- Books can be added and retrieved
- Members can be registered
- Books can be issued within limits
- Books can be returned successfully
- Search finds correct books
- Availability counts are accurate
- Data persists after restart
- Invalid operations are rejected
- Dates are recorded correctly
